# Караоке-сервис для музыкального лото

Загружаете трек → сервис сам распознаёт текст с таймингами (через Whisper API)
→ показывает синхронизированный караоке-текст с возможностью стартовать
с любой секунды ("хук"-момент).

Также умеет подтягивать название, исполнителя, альбом, обложку и текст песни,
а ещё скачивать сам трек списком до 60 штук за раз — по образцу
[spotDL](https://github.com/spotDL/spotify-downloader), только источник
аудио сейчас в первую очередь Яндекс.Музыка, а YouTube — запасной вариант
(см. «Скачивание аудио» ниже, там же объяснение почему). См. разделы
«Автозаполнение метаданных» и «Массовая загрузка треков» ниже.

## Развёртывание (один раз, ~10-15 минут)

### 1. Получите ключ OpenAI
- Зарегистрируйтесь на https://platform.openai.com
- Создайте API-ключ: https://platform.openai.com/api-keys
- Пополните баланс (распознавание стоит центы за минуту аудио — проверьте актуальные цены на странице https://openai.com/api/pricing)

### 2. (Опционально) Получите ключи Spotify для автозаполнения
Нужно только если хотите искать треки и подтягивать название/исполнителя/альбом/обложку одной кнопкой.
- Зайдите на https://developer.spotify.com/dashboard и войдите под своим Spotify-аккаунтом
- Create app → любое название и описание, Redirect URI можно указать любой (например `http://localhost:3000`) — он не используется
- В настройках приложения возьмите `Client ID` и `Client Secret`

### 3. Разверните на Render.com (бесплатный тариф подходит)

Есть два варианта — выбирайте в зависимости от того, нужна ли вам массовая
загрузка треков с YouTube (см. ниже). Если нужна — используйте **вариант Б**,
т.к. для скачивания с YouTube требуется yt-dlp, а его нет в обычном
Node-окружении.

**Вариант А — без массовой загрузки (проще):**
1. Зарегистрируйтесь на https://render.com
2. Загрузите эту папку в свой GitHub-репозиторий
3. В Render: New → Web Service → подключите репозиторий, Environment: **Node**
4. Build command: `npm install`
5. Start command: `npm start`

**Вариант Б — с массовой загрузкой (используется готовый Dockerfile):**
1. Зарегистрируйтесь на https://render.com
2. Загрузите эту папку в свой GitHub-репозиторий
3. В Render: New → Web Service → подключите репозиторий, Environment: **Docker**
   (Render сам найдёт `Dockerfile` в корне и соберёт образ с yt-dlp внутри)

В обоих случаях дальше одинаково:

4. В разделе Environment добавьте переменные:
   - `OPENAI_API_KEY` = ваш ключ OpenAI
   - `SPOTIFY_CLIENT_ID` = ваш Client ID (опционально, для автозаполнения)
   - `SPOTIFY_CLIENT_SECRET` = ваш Client Secret (опционально)
5. Deploy — через пару минут получите публичную ссылку вида `https://ваш-сервис.onrender.com`

(Render — просто пример, подойдёт любой хостинг с поддержкой Docker: Railway,
Fly.io, обычный VPS и т.д. Vercel не подходит — там ограничение на размер
загружаемого файла в 4.5 МБ и нет Docker-окружения для yt-dlp.)

### 4. Локальная проверка перед деплоем (по желанию)
```
npm install
cp .env.example .env
# впишите свои ключи в .env
npm start
```
Откройте http://localhost:3000

## Как пользоваться дальше
Просто открываете ссылку на сервис, загружаете трек — текст распознаётся
и синхронизируется автоматически. Повторно тот же трек распознавать не
нужно — тайминги кэшируются в браузере.

## Общий банк треков (с любого устройства)

Раньше банк треков и раунды хранились в браузере (IndexedDB/localStorage) —
из-за этого с другого устройства или браузера сервис выглядел пустым.
Теперь всё общее и лежит на сервере:

- метаданные (название, исполнитель, текст, тайминги синхронизации, раунды,
  категории) — в SQLite-файле, путь задаётся `DB_PATH`;
- сами аудиофайлы и обложки — в S3-совместимом хранилище (переменные `S3_*`,
  см. `env.example`).

**Критично для деплоя через Coolify:** `DB_PATH` должен указывать на
персистентный том (Storages → добавить том, примонтировать в `/app/data`
внутри контейнера) — иначе база будет обнуляться при каждом передеплое.
Без заданных `S3_*` переменных сервис запустится, но сохранение/загрузка
треков в банк будет выдавать ошибку — остальной функционал (поиск,
распознавание и т.д.) не пострадает.

Если в браузере уже есть локально загруженные раньше треки (до этого
перехода) — при первом открытии сервиса (если банк на сервере ещё пустой)
появится предложение перенести их на сервер автоматически.

⚠️ Сейчас у сервиса нет входа/пароля — banк и раунды доступны и
редактируемы всем, у кого есть ссылка. При нескольких администраторах
возможна гонка при одновременном сохранении раундов с разных устройств
(кто сохранил последним — тот и победил) — это сознательное упрощение
первой версии, не полноценная синхронизация в реальном времени.

## Раунды и категории
Игра состоит из трёх раундов (вкладки «Раунд 1/2/3» на странице игры), между
которыми можно переключаться в любой момент, не дожидаясь, пока предыдущий
доиграется — у каждого раунда своя расстановка чисел и свой прогресс.

В банке треков трек можно пометить одной или несколькими категориями-темами
(«90-е», «Саундтреки» и т.д. — поле «Категории трека» в редакторе; по
умолчанию каждый трек попадает в категорию «Общая»). Перед раундом ведущий
на странице игры выбирает категорию и количество чисел (20/30/40/50/60),
жмёт «Разыграть раунд» — сервис один раз случайно выбирает нужное число
треков из этой категории и раскладывает их по числам. Дальше расстановка
фиксирована: чтобы перераздать раунд заново (например, для новой игры),
нажмите «Разыграть раунд» ещё раз — старая расстановка и отметки сыгранного
для этого раунда сотрутся, попросим подтверждение.

Отдельные числа при необходимости можно поправить вручную — кнопка
«Расставить треки» работает как раньше, независимо от категории.

В банке треков есть фильтр по категории и переключатель сортировки «А-Я»
(быстрее найти нужный трек при большом банке).

## Бланки для участников

На странице игры — кнопка «Бланки для участников»: генерирует печатные
бланки (как в лото) для текущего раунда. Каждый бланк — сетка 5×N из
«Артист - Название», собранная случайно ТОЛЬКО из треков, которые реально
расставлены по числам этого раунда (не из всей категории). У каждого
бланка — свой случайный набор/порядок треков.

Заголовок, текст раунда, «Номинация», подпись и (необязательно) ссылка
для QR-кода — свои поля в окне генератора; QR-код рисуется через
бесплатный сервис api.qrserver.com (ссылка уходит туда открытым текстом
при каждом открытии страницы печати — учитывайте это, если ссылка не
предназначена для чужих глаз). Раскладка бланков сохраняется в раунде
(localStorage) — можно закрыть окно печати и открыть сохранённые бланки
заново кнопкой «Открыть сохранённые», без пересборки. Повторная
«Разыграть раунд» стирает сохранённые бланки этого раунда — придётся
сгенерировать заново.

Сама печать — через открывшуюся страницу и кнопку «Печать / Сохранить
как PDF» (используется системный диалог печати браузера, отдельный PDF-
движок не требуется).

## Автозаполнение метаданных (Spotify)
В разделе «Банк треков» появилось поле поиска: вводите исполнителя и
название, жмёте «Найти» и выбираете нужный вариант из результатов —
подставятся название, исполнитель, альбом, обложка (уменьшается и
сохраняется так же, как при ручной загрузке фото) и текст песни (источник —
открытый lyrics.ovh, без ключа). Дальше как обычно: проверяете текст,
жмёте «Авто-синхрон» или «Синхрон вручную», сохраняете трек.

Если ключи `SPOTIFY_CLIENT_ID`/`SPOTIFY_CLIENT_SECRET` не заданы — поиск
просто выдаст ошибку, остальной функционал сервиса не затронут. Текст
песни на lyrics.ovh находится не всегда — если не нашёлся, поле останется
пустым и его можно заполнить вручную, как раньше.

### Скачивание аудио: Яндекс.Музыка → YouTube

С YouTube-скачиванием есть проблема: с 2025 года YouTube всё активнее
блокирует скачивание с серверных/датацентровых IP (ошибки вида
`403 Forbidden`, `DRM protected`, `Requested format is not available`) —
это ограничение на стороне YouTube, независимо от версии yt-dlp или того,
каким «клиентом» он представляется. Поэтому сервис при скачивании аудио
(в «Массовой загрузке» и «Заменить аудио» по текстовому запросу) **сначала
пробует Яндекс.Музыку** — через ту же библиотеку
[yandex-music](https://github.com/MarshalX/yandex-music-api) — и только
если там ничего не нашлось, откатывается на YouTube через yt-dlp. (Если вы
явно выбрали конкретное видео на YouTube через «Искать варианты» — оно
качается напрямую, Яндекс тут не при чём.)

Поиск/метаданные/текст у Яндекс.Музыки в основном доступны и без токена, но
для надёжного **скачивания самого аудио** токен настоятельно рекомендуется —
без него скачивание может не сработать для части треков. Получить токен:
https://ym.marshal.dev/token/#implicit-oauth, вписать в переменную
`YANDEX_MUSIC_TOKEN`.

Требует Docker-деплой (тот же слой, что и yt-dlp — см. раздел деплоя выше),
без него ни один из источников скачивания не сработает, остальной функционал
(ручная загрузка, распознавание текста, синхронизация) не пострадает.

## Массовая загрузка треков

В «Банке треков» есть кнопка «Массовая загрузка»: открывается окно со
списком — одна песня на строку, «Исполнитель – Название» или прямая ссылка
на YouTube, до 60 строк за раз. По каждой строке сервис по очереди:
ищет метаданные в Spotify (название/исполнитель/альбом/обложка), скачивает
аудио (сначала пробует Яндекс.Музыку, при неудаче — YouTube через yt-dlp,
см. «Скачивание аудио» выше), подтягивает текст с lyrics.ovh — и сразу
сохраняет готовый трек в банк. Статус каждой строки виден в списке
(в очереди / скачивается / готово / ошибка), неудачные строки просто
пропускаются, остальные докачиваются.

Для этой функции нужны yt-dlp и Яндекс.Музыка на сервере — она работает
только при Docker-деплое (вариант Б выше). Скачанный с YouTube аудио-
контейнер не перекодируется (браузер и Whisper одинаково хорошо работают
и с m4a, и с webm), поэтому ffmpeg не обязателен, но добавлен в Dockerfile
на будущее.

**Если и Яндекс.Музыка, и YouTube не нашли трек** (бывает — редкие треки,
локальные записи, самодеятельность) — самый надёжный способ добавить его:
через «+ Новый трек» и загрузку аудиофайла вручную (название/исполнителя/
обложку/текст при этом всё равно можно подтянуть через поиск на Spotify —
см. выше).

**Важно (юридически):** скачивание аудио с YouTube — это то же самое, что делает
spotDL: сервис не имеет отношения к официальному Spotify и не скачивает
файлы с самого Spotify. Легальность такого скачивания зависит от трека,
условий использования YouTube и законодательства вашей страны —
ответственность за использование этой функции лежит на вас.

## Ограничения
- Whisper иногда ошибается в словах на быстрых/шумных участках песни — для
  ключевых треков лото стоит один раз пробежаться глазами по результату.
- Размер трека ограничен 50 МБ (настраивается в server.js).
- Каждое распознавание — платный вызов API (порядка нескольких центов за трек).
- Поиск метаданных берёт название/исполнителя/альбом/обложку из Spotify, а
  текст песни — из отдельного бесплатного источника (lyrics.ovh), т.к.
  Spotify не отдаёт тексты через публичный API. Точность текста не гарантирована.
- Массовая загрузка ищет по названию/исполнителю верхний результат на
  YouTube — для редких треков или похожих каверов результат стоит один раз
  проверить на слух. Скачивание больших/длинных файлов (свыше 30 МБ) обрежется отказом.
- Массовая загрузка работает только при деплое через Docker (см. раздел выше) —
  без yt-dlp на сервере эта кнопка будет выдавать ошибку, остальной функционал
  не пострадает.
- Запасной источник (Яндекс.Музыка) использует неофициальную библиотеку,
  которая не имеет отношения к самой компании Яндекс — она может измениться
  без предупреждения, если Яндекс поменяет своё внутреннее API.
