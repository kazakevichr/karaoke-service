# Караоке-сервис для музыкального лото

Загружаете трек → сервис сам распознаёт текст с таймингами (через Whisper API)
→ показывает синхронизированный караоке-текст с возможностью стартовать
с любой секунды ("хук"-момент).

## Развёртывание (один раз, ~10-15 минут)

### 1. Получите ключ OpenAI
- Зарегистрируйтесь на https://platform.openai.com
- Создайте API-ключ: https://platform.openai.com/api-keys
- Пополните баланс (распознавание стоит центы за минуту аудио — проверьте актуальные цены на странице https://openai.com/api/pricing)

### 2. Разверните на Render.com (бесплатный тариф подходит)
1. Зарегистрируйтесь на https://render.com
2. Загрузите эту папку в свой GitHub-репозиторий (или используйте "Upload" в Render, если поддерживается)
3. В Render: New → Web Service → подключите репозиторий
4. Build command: `npm install`
5. Start command: `npm start`
6. В разделе Environment добавьте переменную: `OPENAI_API_KEY` = ваш ключ
7. Deploy — через пару минут получите публичную ссылку вида `https://ваш-сервис.onrender.com`

(Render — просто пример, подойдёт любой хостинг с поддержкой Node.js: Railway, Fly.io, обычный VPS и т.д. Vercel не рекомендуется — там ограничение на размер загружаемого файла в 4.5 МБ, треки не влезут.)

### 3. Локальная проверка перед деплоем (по желанию)
```
npm install
cp .env.example .env
# впишите свой ключ в .env
npm start
```
Откройте http://localhost:3000

## Как пользоваться дальше
Просто открываете ссылку на сервис, загружаете трек — текст распознаётся
и синхронизируется автоматически. Повторно тот же трек распознавать не
нужно — тайминги кэшируются в браузере.

## Ограничения
- Whisper иногда ошибается в словах на быстрых/шумных участках песни — для
  ключевых треков лото стоит один раз пробежаться глазами по результату.
- Размер трека ограничен 50 МБ (настраивается в server.js).
- Каждое распознавание — платный вызов API (порядка нескольких центов за трек).
